效果：和include连用时，直接将其中的子元素添加到merge标签parent中，这样就保证了不会引入额外的层级

如：include对应的layout.xml的根布局是framelayout，而include的parent也是一个framelayout，
那么，如果不使用merge，那么，view的层次就为 ...-framelayout-framelayout-...
但是，如果使用了merge，那么，view的层次就为 ...-framelayout-...

限制：
Merge只能作为XML布局的根标签使用，
当Inflate以merge开头的布局文件时，必须指定一个父ViewGroup，并且必须设定attachToRoot为true。
	方法签名：LayoutInflater#inflate(@LayoutRes int resource, @Nullable ViewGroup root, boolean attachToRoot)
merge不能作为fragment的layout，因为：
	when you inflate a layout file which has the merge tag as its root 
	you must attach it to a valid parent ViewGroup. 
	Attaching it to the container in the onCreateView is incorrect 
		as the View returned by that method will be added by the framework.
	则：上述做法是矛盾的
